# TMS Simulation and Cerebellar Mapping Pipeline

An automated, end-to-end computational pipeline for simulating Transcranial Magnetic Stimulation (TMS) electric fields and projecting them onto high-resolution 2D cerebellar flatmaps. 

This repository acts as a bridge between **SimNIBS** (a Finite Element Method physics solver) and **SUITPy** (a cerebellar atlas and mapping toolbox). It is designed to allow researchers to model and visualize deep-brain magnetic induction thresholds with high computational efficiency and reproducibility.

---

## 1. Project Overview

**Objective:** To model the electric field (E-field) generated by a Double Cone Coil targeting the cerebellum, and to map the resulting spatial induction onto a flattened representation of the cerebellar cortex.

**Core Operations:**
1. **Physics Simulation:** Utilizes SimNIBS to run a Finite Element Method (FEM) calculation of the electric field magnitude (`magnE`) across the MNI152 head mesh.
2. **Real-World Scaling:** Scales the mathematical baseline (1 A/µs) to true stimulator output thresholds (e.g., 70 A/µs).
3. **Flatmap Projection:** Uses SUITPy to project the 3D NIfTI volume onto a 2D cerebellar flatmap using a conservative `nanmean` mathematical projection.

## 2. Repository Architecture

This pipeline is structured to strictly separate source code, configuration, heavy input data, and generated outputs.

    simulation_tms_cerebellum/
    ├── input/
    │   ├── Drakaki_BrainStim_2022/      <-- Coil files (User provided)
    │   └── m2m_MNI152/                  <-- Head mesh & FreeSurfer surfaces (User provided)
    ├── src/
    │   ├── simnibs_script.py            <-- Physics solver module

    │   └── suitpy/                      <-- Final 2D PNG flatmaps
    ├── .gitignore                       <-- Ignores outputs and large meshes

    ├── requirements.txt                 <-- Python dependencies for SUITPy
    ├── run_pipeline.sh                  <-- The main executable orchestrator
    └── README.md                        <-- This document

## 3. System Requirements

This pipeline orchestrates two different software packages that require their own isolated Python environments. 

1. **SimNIBS (v4.6+)**: Must be installed locally. SimNIBS comes with its own "frozen" Python environment containing necessary C++ libraries. Do not attempt to install SUITPy dependencies inside the SimNIBS environment. [Download SimNIBS here.](https://simnibs.github.io/simnibs/build/html/index.html)
2. **Python 3.10+**: Required to host the SUITPy virtual environment.

---

## 4. Installation and Configuration

**Note on Data Staging:** SimNIBS head meshes (`.msh`) and MRI volumes (`.nii.gz`) frequently exceed standard version control file limits. Therefore, the `input/` folder is explicitly ignored by Git. You must populate the input templates locally before running the pipeline.

### Step 4.1: Clone the Repository
    git clone git@github.com:zeilinga-k-s/simulation_tms_cerebellum.git
    cd simulation_tms_cerebellum

### Step 4.2: Establish the Python Environment
Create a virtual environment specifically for the visualization module and install the required dependencies:
    python3 -m venv venv
    source venv/bin/activate
    pip install -r requirements.txt

### Step 4.3: Populate the Input Data
Create an `input/` directory at the root of this project and stage the following templates from your local SimNIBS installation:

1. **The Head Mesh:** Copy the `m2m_MNI152/` directory (usually found in the SimNIBS `m2m_examples` folder) into `input/`. Ensure the directory name retains its exact capitalization.
2. **The Coil Model:** Copy the coil directory (e.g., `Drakaki_BrainStim_2022/`, found in `simnibs_env/lib/python3.X/site-packages/simnibs/resources/coil_models/`) into `input/`.

### Step 4.4: Configure Software Paths
Open the `config.sh` file and replace the example paths with the exact absolute locations of your SimNIBS Python executable and your newly created SUITPy Python executable.

    # Example config.sh
    simnibs_python_executable="/path/to/your/SimNIBS-4.6/simnibs_env/bin/python3"
    suitpy_python_executable="/path/to/your/simulation_tms_cerebellum/venv/bin/python"

## 5. Execution

Once your inputs are staged and your configuration is set, execute the master orchestrator script from your terminal:

    bash run_pipeline.sh

**Pipeline Workflow:**
1. **Pre-flight Check:** Automatically detects and clears previous `output/` files for the specified subject to prevent physics engine conflicts.
2. **Physics Solver:** Triggers `src/simnibs_script.py` to calculate the spatial electric field magnitude.
3. **Standardization:** Isolates the correct `.nii.gz` volume from the SimNIBS output array.
4. **Visualization:** Triggers `src/suitpy_script.py`, scales the baseline field by the specified stimulator power, and flattens the 3D data onto the SUIT cerebellar atlas.
5. **Review:** Opens the final high-resolution flatmap.

---

## 6. Output Specifications

Generated files are organized into the following directories:

* **`output/simnibs/`**: Contains the raw 3D `.msh` files (viewable in Gmsh) and the scaled `simulation_output.nii.gz` volume.
* **`output/suitpy/`**: Contains the publication-ready, high-resolution `.png` flatmaps. The colour mapping utilizes a `nanmean` projection to provide a mathematically conservative estimate of the transcortical induction field.

---

## 7. Operating System Compatibility

This pipeline is orchestrated using Bash (`.sh`) scripts and is natively optimized for **Linux** environments.

* **macOS:** Operates natively via the terminal. Ensure paths in `config.sh` reflect macOS application structures (e.g., `/Applications/SimNIBS-4.6/...`).
* **Windows:** Standard Windows Command Prompt cannot natively execute Bash scripts. Users must install the **Windows Subsystem for Linux (WSL 2)**, install SimNIBS and Python within the Ubuntu subsystem, and run the pipeline from the WSL terminal.
